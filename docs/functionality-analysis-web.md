# rss-impact-web 功能分析

## 项目概述

rss-impact-web 是一个基于 Vue.js 的前端应用，是 RSS Impact 系统的前端部分。该系统是一个支持 Hook 的 RSS 订阅工具，主要用于管理和处理 RSS 订阅、文章内容以及通过 Hook 机制进行自动化通知和处理。

## 业务功能划分

### 1. 用户认证与管理

**核心功能：否**

#### 页面

-   **登录页面** (`/login`): 用户登录
-   **注册页面** (`/register`): 新用户注册
-   **个人中心** (`/personal`): 用户个人信息管理

#### 接口

-   用户登录认证
-   用户注册
-   用户信息查询/更新
-   密码修改

### 2. RSS 订阅管理

**核心功能：是**

#### 页面

-   **订阅管理** (`/feed`): RSS 源的添加、编辑、删除和管理
-   **分类管理** (`/category`): RSS 订阅的分类组织
-   **文章管理** (`/article`): 订阅内容的查看和管理

#### 接口

-   RSS Feed 的 CRUD 操作
-   分类的 CRUD 操作
-   文章的查询、标记已读等操作

### 3. Hook 配置与通知

**核心功能：是**

#### 页面

-   **Hook 管理** (`/hook`): 配置自动化处理规则和 Hook
-   **Webhook 和通知日志** (`/webhook-log`): 查看 Hook 执行日志和通知记录

#### 接口

-   Hook 配置的 CRUD 操作
-   Webhook 日志查询
-   通知发送记录

### 4. 数据查询与分析

**核心功能：否**

#### 页面

-   **自定义查询** (`/custom-query`): 高级数据查询功能
-   **使用统计** (`/admin/daily-count`): 系统使用统计 (管理员权限)

#### 接口

-   自定义查询执行
-   统计数据获取

### 5. 系统配置

**核心功能：否**

#### 页面

-   **代理管理** (`/proxy-config`): 网络代理配置
-   **资源管理** (`/resource`): 系统资源管理

#### 接口

-   代理配置 CRUD
-   资源管理 CRUD

### 6. 后台管理 (管理员权限)

**核心功能：否**

#### 页面

-   **用户管理** (`/admin/user`): 用户账户管理
-   **系统管理** (`/admin/system`): 系统设置和配置

#### 接口

-   用户管理 CRUD
-   系统配置管理

### 7. 其他页面

**核心功能：否**

#### 页面

-   **主页** (`/`): 系统概览和快速入口
-   **文档** (`/docs`): 系统使用文档
-   **关于** (`/about`): 项目介绍和信息
-   **404 页面** (`/404`): 错误页面

## 核心功能总结

1. **RSS 订阅管理**: 系统核心，提供 RSS 源订阅、分类和文章管理功能
2. **Hook 机制**: 自动化处理和通知的核心，支持自定义 Hook 配置和执行
3. **通知系统**: 通过 Webhook 和日志记录实现内容推送和状态跟踪

## 技术架构

-   **前端框架**: Vue 3 + TypeScript
-   **UI 组件**: Element Plus + Avue
-   **状态管理**: Pinia
-   **路由**: Vue Router
-   **HTTP 客户端**: Axios (通过 Swagger 生成的 API)
-   **构建工具**: Vite
-   **样式**: SCSS + Normalize.css

## 接口说明

系统通过 Swagger 生成的 TypeScript API 与后端通信，主要接口包括：

-   认证相关接口
-   CRUD 操作接口 (用于各类数据管理)
-   查询和统计接口
-   Hook 执行和日志接口

所有接口都通过统一的响应拦截器处理错误和认证状态。
